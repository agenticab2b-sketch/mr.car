# Typography (Material 3) — Web

## Цель
Сформировать типографику сайта как систему (шрифты + иерархия + правила применения), которая:
- выглядит аккуратно и современно,
- читается на **mobile / tablet / desktop**,
- сохраняет иерархию (H1 > H2 > H3 > Body),
- подходит под язык(и) и тон проекта.

## Когда использовать
Используй этот skill, когда нужно:
- выбрать шрифты для сайта,
- задать шкалу заголовков и основного текста,
- настроить читабельность для мобилы/планшета/десктопа,
- проверить, что текст не “убивает” визуальный стиль.

## Входные данные (минимум)
Если чего-то нет — сделай **не более 1–2 допущений** и явно укажи строкой `Assumption: ...`.
- тип проекта (автосервис / B2B / магазин / личный бренд)
- язык(и): RU / EE / EN
- тон: строго / дружелюбно / премиально / технично
- ограничения (например: нужна кириллица; без декоративных шрифтов; максимальная читаемость на мобиле)
- ключевые блоки (главная / услуги / прайс / форма / FAQ)

## Жёсткие правила
1) **Строго 1–2 семейства шрифтов** (одно на всё или заголовки+текст).
2) **Семейства шрифтов не меняются между устройствами**. Меняются только размеры/межстрочный/интервалы.
3) Ответ обязан включать матрицу **Mobile / Tablet / Desktop** для каждого уровня текста.
4) Если есть RU — шрифты должны **нормально поддерживать кириллицу**.
5) Структуру ответа из `resources/TEMPLATE.md` **нельзя сокращать или переставлять**.

## Decision tree (быстрое решение)
- Если язык включает **RU** → выбирай шрифты с корректной кириллицей (без компромиссов).
- Если тон **премиально/строго** → избегай “игривых” гарнитур и резких контрастов начертаний.
- Если сайт контентный (много текста) → приоритет читабельности (Body/Small/Caption особенно).
- Если входных данных мало → 1–2 допущения и пометка `Assumption:` (не молча).
