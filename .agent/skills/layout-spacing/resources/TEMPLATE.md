## A) Стратегия компоновки (1–3 строки)
- Какой характер layout выбран (строго/воздушно/плотно) и почему это подходит проекту.
- Что будет “несущей конструкцией”: ширина контента, колонки, секции, карточки.

---

## B) Grid & page frame (обязательная матрица Mobile / Tablet / Desktop)
Для каждого устройства укажи:
- ширина контента (или правило ширины)
- боковые поля (page margins)
- базовая колонночность (если используешь)
- правило: где контент “фиксируется”, а где может быть шире (например hero/фон)

Формат:
### Mobile
- Content width: ...
- Page margins: ...
- Columns: ...
- Rule: ...

### Tablet
- Content width: ...
- Page margins: ...
- Columns: ...
- Rule: ...

### Desktop
- Content width: ...
- Page margins: ...
- Columns: ...
- Rule: ...

---

## C) Spacing scale (единая шкала)
Дай шкалу отступов (пример: 4/8/12/16/24/32/48/64) и правила:
- какие шаги “мелкие” (внутри компонентов)
- какие “средние” (между блоками внутри секции)
- какие “крупные” (между секциями)

---

## D) Vertical rhythm (ритм секций) — минимум 6 правил
Коротко и конкретно:
- отступы между секциями
- отступы внутри секции (заголовок → контент)
- правила для списков/карточек
- правила для прайса/таблиц
- правила для форм (между полями, группами, кнопкой)
- правило “когда можно плотнее, когда нельзя”

---

## E) Responsive rules (правила адаптации)
Опиши:
- что меняется при переходе mobile → tablet → desktop (колонки, плотность, ширины)
- какие элементы обязаны оставаться в одной колонке на mobile
- как вести себя таблицам/прайсам (перенос, карточный вид, прокрутка — выбирай и объясни)

---

## F) Запрещённые ошибки (7–12 пунктов)
Формат: “не делай так”, коротко и конкретно.
Примеры: случайные отступы; разные поля на страницах; слишком широкие тексты; тесные формы; хаос между секциями.

---

## G) Мини-чеклист перед выпуском (10 пунктов)
Чеклист должен позволять быстро проверить:
- единая шкала отступов
- одинаковые поля страницы
- предсказуемый ритм секций
- адекватная ширина текста
- нормальная адаптация прайса/таблиц/форм
