# Layout & Spacing (Material 3) — Web

## Цель
Сформировать систему **сетки и отступов** для сайта, чтобы:
- страница выглядела собранно и “дорого”,
- был ровный ритм между секциями и элементами,
- всё адаптировалось под **mobile / tablet / desktop** без хаоса,
- компоненты не “плавали” по ширинам и отступам.

## Когда использовать
Используй этот skill, когда нужно:
- определить ширину контента, поля страницы и сетку,
- задать шкалу отступов (spacing scale),
- описать правила адаптации на mobile/tablet/desktop,
- проверить консистентность расстояний и компоновки.

## Входные данные (минимум)
Если чего-то нет — сделай **не более 1–2 допущений** и явно укажи строкой `Assumption: ...`.
- тип проекта (автосервис / B2B / магазин / личный бренд)
- тип страниц (главная / услуги / прайс / контакты / FAQ)
- плотность контента: мало текста / средне / много
- “стиль компоновки”: строго/воздушно/плотно
- ключевые блоки: hero, преимущества, услуги, прайс/таблица, форма, FAQ

## Жёсткие правила
1) **Одна шкала отступов** на весь сайт (никаких “здесь 17px, а тут 23px”).
2) Сетка и отступы обязаны быть описаны для **mobile / tablet / desktop**.
3) Ширины, поля, ритм секций и расстояния внутри карточек/форм — должны подчиняться одной системе.
4) Нельзя “рисовать отступы на глаз”. Все значения — из шкалы.
5) Структуру ответа из `resources/TEMPLATE.md` **нельзя сокращать или переставлять**.

## Decision tree
- Если контента много → приоритет читабельности: ширина текста ограничена, больше вертикального ритма.
- Если “прайс/таблицы/списки” важны → закладывай строгую колонночность и стабильные отступы.
- Если стиль “плотно” → уменьши шаги шкалы, но не нарушай системность.
- Если входных данных мало → 1–2 допущения и `Assumption:` (не молча).
