# Shape & Depth (Material 3) — Web

## Цель
Сформировать систему **форм (скругления)** и **глубины (elevation/слои)** для сайта, чтобы:
- кнопки, карточки, формы и секции выглядели согласованно,
- интерфейс был “цельным продуктом”, а не набором случайных блоков,
- глубина использовалась по делу, без визуального шума.

## Когда использовать
Используй этот skill, когда нужно:
- выбрать и закрепить скругления для всех элементов,
- определить, где уместна глубина (карточки, всплывающие элементы),
- согласовать “характер” (строго/премиально/индастриал/мягко),
- устранить хаос “тени и радиусы везде разные”.

## Входные данные (минимум)
Если чего-то нет — сделай **не более 1–2 допущений** и явно укажи строкой `Assumption: ...`.
- тип проекта (автосервис / B2B / магазин / личный бренд)
- тон: строго / дружелюбно / премиально / технично / “брутально-индастриал”
- предпочтение по “характеру формы”: более угловато / более мягко
- ключевые элементы: кнопки, карточки, поля ввода, модалки/меню, таблицы/прайс
- запреты: (например: “без сильных теней”, “минимализм”, “не выглядит как игрушка”)

## Жёсткие правила
1) **Одна шкала скруглений** на весь сайт. Никаких “тут 6, там 18, потому что красиво”.
2) **Скругления выбираются по типу элемента** (кнопки/карточки/поля/секции) и не прыгают между страницами.
3) Глубина (elevation) — это инструмент иерархии: используется, чтобы выделить слой/важность, а не “для украшения”.
4) Если проект “строгий/премиальный/индастриал” — избегай чрезмерно “пухлых” форм и жирных теней.
5) Структуру ответа из `resources/TEMPLATE.md` **нельзя сокращать или переставлять**.

## Decision tree
- Если стиль “строго/индастриал” → радиусы меньше, глубина сдержанная, акцент на чётких краях.
- Если стиль “дружелюбно/мягко” → радиусы больше, глубина мягче, но без перебора.
- Если много карточек/каталога/прайсов → особенно важно закрепить единый радиус карточек и правила глубины.
- Если входных данных мало → 1–2 допущения и `Assumption:` (не молча).
